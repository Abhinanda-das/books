# Системы автоматического «конкурентного техно-мониторинга»: что уже есть и из чего собрать своё

## TL;DR
- **Готового продукта «под ключ», который именно так, как вы описали (вотчлист → периодический скан → deep-research отчёт → оценка по вашим критериям «появился ли достойный конкурент»), не существует — но все кирпичики уже есть, и одно open-source решение (`duanyytop/agents-radar`) подходит очень близко.** Это ситуация «частично изобретено / легко собирается», а не «всё уже сделано».
- Коммерческие платформы конкурентной разведки (Crayon, Klue, Feedly Market Intelligence, Contify) умеют мониторить и AI-суммаризировать сигналы, но заточены под бизнес-конкурентов и стоят дорого (Feedly Market Intelligence — от $1 600/мес, то есть от $19 200/год), а dev/open-source экосистемы (GitHub, Hugging Face, PyPI/npm) покрывают слабо.
- Самый реалистичный путь для вас как для vibe-кодера на macOS, уже использующего Claude + MCP: собрать связку из бесплатных источников-мониторов (GitHub-релизы/тренды, Hugging Face trending, NewReleases.io, RSS) + расписания (GitHub Actions / Claude Routines) + Claude как «слой deep-research и оценки по критериям» через готовые MCP-серверы (GitHub, Hugging Face, Hacker News, RSS).

## Key Findings

**1. Точного «персонального техно-скаута с оценкой по критериям» как коробочного продукта нет.** Отдельные части пайплайна (вотчлист, мониторинг, deep-research, скоринг) существуют по отдельности и хорошо документированы, но ни один продукт не объединяет: (а) вотчлист именно dev/OSS-инструментов, (б) мультиисточниковый скан, и (в) строгую оценку по заданным пользователем критериям, которая прямо выдаёт «появилась достойная альтернатива». Это и есть реальная «щель» (gap).

**2. Ближайший к вашей идее open-source проект — `duanyytop/agents-radar`** (GitHub). Это ежедневный AI-дайджест, который тянет данные из ~10 источников (GitHub trending/releases, arXiv, Hacker News, Hugging Face, Product Hunt, Dev.to, Lobste.rs + RSS), вотчлист настраивается через `config.yml` без правки кода, LLM (Anthropic/OpenAI/др.) отфильтровывает нерелевантное, классифицирует по категориям и извлекает «сигналы трендов», делает кросс-экосистемное сравнение. Выдаёт Markdown + GitHub Issues + статический сайт на GitHub Pages + RSS + MCP-сервер, запускается по расписанию через GitHub Actions. Минус: скоринг качественный/нарративный, а не строгая числовая рубрика по вашим критериям.

**3. Явный «скоринг по критериям» уже реализован в `MorganREN/research-agent-radar` (PaperFlow.AI)** — мультиагентный пайплайн, где LLM оценивает каждую находку по шкале 1–10 относительно заданных пользователем интересов, с расписанием раз в 6/12/24 ч. Но он покрывает только академические статьи (arXiv/ScienceDirect), а не GitHub/HF/Product Hunt. Ещё один похожий — `myzhao0114-del/Omni-research-radar` (скилл для Claude/Codex, ранжирует статьи + GitHub-проекты по своим критериям).

**4. Коммерческие платформы конкурентной разведки мощные, но не про вас.** Crayon, Klue, Contify, Feedly Market Intelligence, AlphaSense, Kompyte автоматизируют мониторинг конкурентов, AI-суммаризацию и рассылки, но: (а) заточены под отделы продаж/маркетинга и бизнес-конкурентов, а не под dev-инструменты и open-source; (б) дорогие (Feedly Market Intelligence — от $1 600/мес; AlphaSense — медианный контракт $18 375/год при диапазоне $12K–$51K по данным Vendr из 34 реальных закупок). Для отслеживания «появился ли MCP-коннектор для Obsidian RAG» они не предназначены.

**5. Для dev/OSS-экосистем есть отдельные специализированные бесплатные/дешёвые инструменты** мониторинга релизов и трендов: NewReleases.io (релизы GitHub/GitLab/PyPI/npm/Docker/etc.), Libraries.io (открытые пакеты и зависимости), star-history.com и daily-stars-explorer (динамика звёзд), GitHub Trending (скорость набора звёзд), Hugging Face trending (загрузки/лайки моделей). Это «датчики», но без слоя оценки и без единого отчёта.

## Details

### PART A — Готовые / коммерческие / SaaS-продукты

Эти платформы делают «конкурентную/рыночную разведку» и AI-алертинг, но в основном для бизнес-конкурентов:

- **Crayon (crayon.co)** — мониторит конкурентов, шлёт приоритезированные инсайты в инбокс, AI-суммаризация новостей, battlecards, интеграции со Slack/Teams/Salesforce. Цена корпоративная (по запросу). Покрытие dev/OSS: слабое. Кастомные критерии оценки: ограниченно (приоритезация тем).
- **Klue** — all-in-one конкурентная разведка, в 2026 добавили «Compete Agent» (AI-агент с инсайтами в воркфлоу сделок). Цена корпоративная, по запросу (публичных тарифов нет; по пользовательским оценкам — порядка $16 000–30 000/год, но это неподтверждённая оценка). Не про dev-инструменты.
- **Contify** — новостно-ориентированная разведка, в 2025 запустили агентный движок «Athena» со знание-графами. Кастомные дашборды. Цена по запросу.
- **Feedly Market Intelligence (feedly.com/market-intelligence)** — наиболее близкая по духу: «AI Feeds» на 10 000+ AI-моделей, «Emerging Trends Dashboard» (ловит «слабые сигналы»), Insights Cards, авто-рассылки, API, интеграции со Slack/Teams. Можно настраивать ленты по компаниям/технологиям/темам. **Цена (при годовой оплате): Standard — $1 600/мес (100 AI Feeds, до 10 мест); Advanced — $2 400/мес (200 AI Feeds, Ask AI, API, SSO), то есть от $19 200/год.** Индивидуальный тариф Feedly Pro+ с AI-ассистентом Leo стоит $12,99/мес при помесячной оплате или $8,25/мес ($99/год) при годовой — это единственный тариф для одного пользователя с Leo AI, но **без** полноценных AI Feeds, дашбордов трендов и API. Это самый реалистичный коммерческий вариант, если бюджет позволяет, но он про общие технологии/компании, а не про точечный мониторинг GitHub-репозиториев и MCP-коннекторов.
- **AlphaSense** — институциональная разведка (отчёты, транскрипты), очень дорого (медианный контракт ~$18 375/год по данным Vendr), не для вашей задачи.
- **Visualping (visualping.io)** — простой мониторинг изменений на веб-страницах с AI-флагом «важно/неважно» и кратким резюме изменения; от ~$13/мес. Хорош как «датчик» на конкретные страницы (например, страница релизов или прайс), но без понимания экосистем и без скоринга.
- **Specialized release/awareness-сервисы:** The Git Gazette (gitgazette.com) — AI-«газета» по GitHub-репозиториям, ранжирует тренды по звёздам/скорости/моментуму (бесплатные дайджесты + платный API); Releases Tracker на GitHub Marketplace — авто-подписка на релизы ваших starred-репозиториев, бесплатно; GitHubTracker (githubtracker.com) — аналитика по репозиторию. Все они покрывают dev/OSS, но **не делают кастомный скоринг и не формулируют «появилась лучшая альтернатива»**.

### PART B — Open-source кирпичики для самостоятельной сборки

**(1) Вотчлисты / мониторинг релизов и репозиториев:**
- **NewReleases.io** — отслеживает релизы из GitHub, GitLab, PyPI, npm, Docker Hub, Maven, Cargo, Packagist и др.; уведомления на e-mail/Slack/Discord/Teams/webhook; есть API и CLI (open-source клиент). Бесплатный тариф есть.
- **Libraries.io** (open-source, librariesio/libraries.io) — индексирует 10+ млн пакетов из ~33 пакетных менеджеров, показывает зависимости, новые/трендовые/популярные пакеты; есть API и открытый датасет; помогает находить альтернативные библиотеки.
- **GitHub Trending** — лента репозиториев по скорости набора звёзд (звёздная «velocity»). Официального API нет (только HTML-страница), но есть скраперы (например, Apify-актор с «watchlist mode», который выдаёт только репозитории, у которых изменился ранг/звёзды — фактически чистый diff-фид «что набирает обороты»).
- **Banditore** (self-hosted) — собирает новые релизы из ваших starred-репозиториев в персональный RSS.
- **Hugging Face trending** — модели/датасеты/спейсы по загрузкам и лайкам.

**(2) Сигналы adoption / популярности:**
- **star-history.com** и **emanuelef/daily-stars-explorer** (open-source, Docker) — динамика звёзд по дням, сравнение репозиториев. Ключевая идея: важна не сумма звёзд, а **velocity** (скорость прироста) — именно она показывает зарождающегося конкурента.
- **Libraries.io** — `dependents_count` и зависимости как сигнал реального использования (а не просто звёзд).
- **Hugging Face** — тренды загрузок/лайков моделей как сигнал принятия сообществом.
- Метрики GitHub: star velocity, отношение звёзд к форкам, частота коммитов, активность issue/PR.

**(3) Расписание / автоматизация:**
- **GitHub Actions (cron)** — бесплатно для публичных репо, дёшево для приватных; де-факто стандарт для «запусти скрипт/агента по расписанию». Есть готовый паттерн «AI news bot»: RSS → LLM (Claude/DeepSeek) → e-mail, запуск по `cron: '0 0 * * *'`, стоимость ~$0.
- **Claude Routines** (research preview с 14 апреля 2026, на платных планах Pro/Max/Team/Enterprise) — облачные запланированные запуски агента Claude (ежечасно/ежедневно/еженедельно, и по GitHub-вебхукам), не требуют держать ноут включённым. Прямо названы под «competitor monitoring», «news digest», «social listening». Создаются на claude.ai/code/routines или командой `/schedule`.
- **Claude Code Loops / cron_create** — встроенное расписание в самом Claude Code (но loops живут только в рамках сессии; для надёжности — GitHub Actions или Routines).
- **n8n** (open-source, self-host) — визуальные воркфлоу; есть готовые шаблоны конкурентного анализа (Exa.ai + Notion + AI-агенты; GPT-4 + Apify + Google Docs) и встроенные «Evaluations» с LLM-as-a-Judge и кастомными метриками. Хорош как «дирижёр».
- **changedetection.io** (open-source, self-host, Docker) — мониторинг изменений веб-страниц и RSS/XML с уведомлениями; альтернатива Visualping. **Huginn** — более мощный self-hosted «агент-конструктор» (читает веб, следит за событиями, действует). Zapier/Make — облачные no-code альтернативы.

**(4) AI deep-research + слой оценки:**
- **GPT Researcher (assafelovic/gpt-researcher, ~27,3 тыс. звёзд на июнь 2026, Apache-2.0)** — автономный агент, который планирует под-вопросы, параллельно обходит 20+ источников, дедуплицирует и пишет отчёт (в среднем 5–6 страниц в форматах PDF/Docx/Markdown) с цитатами; работает с любым LLM; **поддерживает MCP-интеграцию** (можно подключить GitHub-MCP как источник наряду с веб-поиском); занял 1-е место в бенчмарке DeepResearchGym от CMU. Есть отдельный gptr-mcp (MCP-сервер) для Claude.
- **n8n Evaluations / LLM-as-a-Judge** — слой скоринга: прогнать находки через судью-LLM, оценить по вашим критериям (новые возможности, скорость, точность, моментум), задать порог-алерт. Рекомендация из практики n8n: лучше задавать **сравнительные вопросы** («содержит ли новый вывод корректную информацию?»), чем абсолютные оценки 1–10, плюс добавлять детерминированные метрики (звёзды, загрузки) — это даёт устойчивый скоринг.
- **DeepEval, Arize Phoenix (open-source), Promptfoo** — фреймворки оценки LLM-выводов (50+ метрик, LLM-as-judge), можно переиспользовать как «оценочный движок».

**(5) MCP-серверы, релевантные мониторингу (вы уже используете Claude + MCP):**
- **Официальный Hugging Face MCP (huggingface.co/mcp)** — поиск моделей/датасетов/спейсов/статей, метаданные (загрузки, ссылки); бесплатный HF-токен достаточен. Плюс community `kukapay/hf-trending-mcp` (трендовые модели/датасеты/спейсы с загрузками и лайками) и `davanstrien/hub-semantic-search-mcp` (семантический поиск + `get_trending_models`).
- **Hacker News MCP** — несколько реализаций (`erithwik/mcp-hn`, `devabdultech/hn-mcp` с поиском через Algolia, `rawveg/hacker-news-mcp` с анализом трендов).
- **RSS MCP** — серверы для чтения RSS/Atom (например, на Hugging Face Spaces).
- **GitHub MCP** — официальный сервер Model Context Protocol для GitHub (релизы, репозитории, issues), используется в т.ч. внутри GPT Researcher.
- Это значит, что Claude может сам ходить в эти источники и сразу применять ваши критерии оценки — без написания интеграционного кода.

**(6) Tech-radar / визуализация решений:**
- **ThoughtWorks Technology Radar** — методология «Adopt / Trial / Assess / Hold (Caution)» в квадрантах (Techniques / Platforms / Tools / Languages & Frameworks); публикуется дважды в год.
- **thoughtworks/build-your-own-radar** (open-source, AGPL) — генерирует интерактивный радар из Google Sheet / CSV / JSON; можно хостить приватно.
- Альтернативы: **zalando/tech-radar** (авто-расстановка точек, статический сайт), **qiwi/tech-radar** (полностью автоматический генератор из json/csv/yml через CLI/npx), AOEpeople, dprgarner/tech-radar-generator. Это удобный способ визуализировать «что я использую сейчас vs. что появляется».
- Есть даже n8n-шаблон «AI-powered tech radar advisor» (Google Sheet → SQL + векторная БД → агент-роутер SQL/RAG + guardrail) — близко к идее «спрашивать у радара».

### PART C — «А не изобретаю ли я велосипед?»

Честный ответ: **скорее нет, чем да — это пространство занято лишь частично.**
- Building blocks все есть; несколько open-source проектов уже объединяют вотчлист + мультиисточниковый скан + LLM-анализ + Markdown/статический сайт (прежде всего **`duanyytop/agents-radar`**).
- **`MorganREN/research-agent-radar`** независимо доказывает, что «LLM оценивает находки по шкале относительно ваших критериев» — рабочая и реализованная штука (но пока на академических статьях).
- **`myzhao0114-del/Omni-research-radar`** — лёгкий скилл-радар для Claude/Codex, который ранжирует находки.
- Также стоит упомянуть `LearningCircuit/local-deep-research` (~4 000 звёзд, self-hosted, мультиисточниковый research-ассистент с оценкой репутации источников, но он query-driven, а не мониторинг вотчлиста) и кураторские списки awesome-ai-news, где собраны десятки дайджест-инструментов (meridian, auto-news, TrendRadar, customize-arxiv-daily с кастомными промптами + локальный Ollama).
- **Чего нет:** ни один проект или отполированный SaaS не объединяет все три условия сразу — (1) вотчлист именно dev/OSS-инструментов, (2) мультиисточниковый скан, и (3) строгий скоринг по вашим критериям, прямо выдающий «появился достойный конкурент/лучшая альтернатива». agents-radar делает качественные «сигналы трендов» (не строгую рубрику); research-agent-radar делает рубричный скоринг (но только статьи); SaaS-инструменты делают уведомления/аналитику (без скоринга).

*Примечание по источнику:* в одном из агрегаторов GitHub-тем упоминается проект **«Recoleta»** — «local-first AI research radar, превращающий статьи, обсуждения и фиды в Markdown-брифы и статический сайт». Прямую страницу репозитория подтвердить отдельным поиском **не удалось**, поэтому относитесь к этому названию с осторожностью; по описанию он функционально совпадает с agents-radar и arXiv-дайджест-инструментами.

## Recommendations

Три реалистичных пути, по нарастанию усилий и гибкости:

**Путь 1 — «Минимум усилий, использовать готовое» (старт за выходные).**
- Подпишитесь на «датчики»: NewReleases.io (релизы ваших инструментов из GitHub/PyPI/npm/Docker), GitHub Trending по нужным языкам, Hugging Face trending, плюс 1–2 дайджеста (TLDR AI, Product Hunt). Для отслеживания конкретных страниц (например, «появился ли MCP-коннектор для Obsidian RAG») — changedetection.io или Visualping.
- Когда нужен разовый «глубокий разбор» — попросите Claude (с подключёнными MCP: GitHub, Hugging Face, Hacker News, RSS) собрать отчёт и оценить находки по вашим критериям.
- **Плюсы:** почти ноль настройки, бесплатно/дёшево. **Минусы:** нет единого автоматического отчёта со скорингом; вы сами сводите.
- *Порог перехода дальше:* если вручную сводить сигналы надоедает чаще ~раза в неделю — переходите к Пути 2.

**Путь 2 — «Собрать open-source пайплайн» (рекомендуемый для вас).**
- Возьмите **`duanyytop/agents-radar`** как основу (он почти про это: вотчлист в `config.yml`, источники GitHub/HF/HN/Product Hunt/RSS, LLM-анализ, вывод в Markdown/сайт/RSS/MCP, запуск через GitHub Actions). Подключите ваш Claude-ключ.
- Добавьте слой **явной оценки по критериям**: либо через n8n Evaluations (LLM-as-a-Judge + детерминированные метрики вроде star-velocity и загрузок HF), либо допишите в промпт agents-radar рубрику («новые возможности / скорость / точность / моментум принятия сообществом → вердикт: стоит ли это вашему текущему стеку»).
- Расписание: GitHub Actions (бесплатно) или Claude Routines.
- **Плюсы:** очень близко к вашему ТЗ, бесплатно, под полным контролем. **Минусы:** нужно немного повозиться с конфигом и промптами (но это посильно vibe-кодеру).

**Путь 3 — «Кастомный Claude + MCP агент» (максимум под вас).**
- Claude Routines (или GitHub Actions + `claude -p`) по расписанию запускает агента; агент через MCP-серверы (GitHub, Hugging Face, HN, RSS) собирает свежие сигналы по вашему вотчлисту, при необходимости вызывает GPT Researcher (через gptr-mcp) для deep-research, затем применяет ваши критерии и пишет отчёт прямо в Obsidian (у вас уже есть Obsidian MCP) с вердиктом «появился ли достойный конкурент».
- Визуализацию «текущий стек vs. новички» сделайте через build-your-own-radar или qiwi/tech-radar.
- **Плюсы:** идеально под вашу формулировку и ваш стек (Claude+MCP+Obsidian). **Минусы:** больше всего сборки и поддержки промптов/логики.

**Общая рекомендация:** начните с Пути 1 (1–2 дня), чтобы понять, какие источники и критерии реально полезны, и сразу переходите к Пути 2 на базе `agents-radar`, дописав слой скоринга по критериям. К Пути 3 переходите, только если хотите, чтобы отчёты автоматически попадали в Obsidian и были полностью «вашими».

**Про критерии «что значит лучше».** Определите 4–6 измеримых критериев заранее (например: новые возможности/подход; скорость/эффективность; точность/качество; momentum принятия — star-velocity, прирост загрузок HF, упоминания на HN/Product Hunt; зрелость — частота релизов, число зависимых проектов; совместимость с вашим стеком/MCP). Применяйте их **после** сбора находок (скоринг постфактум) — это надёжнее, чем фильтровать на входе, и совпадает с практикой LLM-as-a-Judge.

## Caveats
- **Звёзды и тренды ≠ качество.** Star-velocity показывает внимание сообщества «прямо сейчас», но трендовый репозиторий может оказаться экспериментом выходного дня, а не зрелой библиотекой. Используйте звёзды вместе с сигналами реального использования (зависимости в Libraries.io, частота релизов, загрузки HF).
- **GitHub Trending не имеет официального API** — только HTML-страница; популярные языки (JS/Python/TS) доминируют, нишевые экосистемы видны хуже.
- **Claude Routines — это research preview** (с 14 апреля 2026): поведение, лимиты и API могут меняться; на GitHub-вебхуки есть почасовые лимиты. Loops в Claude Code не переживают перезапуск/закрытие сессии — для надёжного расписания используйте GitHub Actions или Routines.
- **LLM-скоринг субъективен и недетерминирован.** Один и тот же промпт может давать разные оценки; добавляйте детерминированные метрики и сравнительные (а не абсолютные) вопросы, периодически проверяйте калибровку.
- **Стоимость.** Коммерческие платформы конкурентной разведки (Feedly MI от $1 600/мес, AlphaSense ~$18 375/год медиана) почти наверняка избыточны и не покрывают dev/OSS-фокус; для вашей задачи open-source + Claude-подписка дешевле и точнее.
- **«Recoleta»** как конкретный проект не удалось подтвердить прямым поиском — не закладывайтесь на это название, пока сами не найдёте рабочий репозиторий.
- Названия некоторых продуктов из общих обзоров (например, «Releasly») подтвердить как реально существующие отдельные продукты не удалось; большинство «release»-сервисов оказались инструментами для публикации СВОИХ чейнджлогов (LaunchNotes, Beamer, AnnounceKit), а не для входящего скаутинга.
